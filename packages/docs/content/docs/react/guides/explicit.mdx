title: Explicit
description: Explicitly render the reCAPTCHA widget
---

The explicit implementation of reCAPTCHA allows you to have full control over when and how the reCAPTCHA widget is rendered on your page by script injection.

## Types and Differences

The explicit implementation supports different types of reCAPTCHA:

### v2 Checkbox

The classic checkbox-style reCAPTCHA that requires users to check a box and potentially solve a challenge.

```typescript
interface ReCaptchaV2CheckboxProps {
  sitekey: string;
  theme?: 'light' | 'dark';
  size?: 'normal' | 'compact';
  tabindex?: number;
  onExpired?: () => void;
  onError?: () => void;
  onSuccess?: (token: string) => void;
}
```

### v2 Invisible

A version that runs in the background and only shows challenges when necessary.

```typescript
interface ReCaptchaV2InvisibleProps {
  sitekey: string;
  badge?: 'bottomright' | 'bottomleft' | 'inline';
  onExpired?: () => void;
  onError?: () => void;
  onSuccess?: (token: string) => void;
}
```

### v3

A score-based version that runs in the background and returns a score instead of showing challenges.

```typescript
interface ReCaptchaV3Props {
  sitekey: string;
  action: string;
  onExpired?: () => void;
  onError?: () => void;
  onSuccess?: (score: number) => void;
}
```

## Usage Examples

### Checkbox Example

```tsx
import { ReCaptchaV2Checkbox } from '@hcaptcha/react-hcaptcha';

function MyForm() {
  const handleSuccess = (token: string) => {
    // Handle the verification token
    console.log('Verification successful:', token);
  };

  return (
    <form>
      <ReCaptchaV2Checkbox
        sitekey='your-site-key'
        theme='light'
        size='normal'
        onSuccess={handleSuccess}
        onError={() => console.error('Verification failed')}
        onExpired={() => console.log('Verification expired')}
      />
    </form>
  );
}
```

### Badge Example

```tsx
import { ReCaptchaV2Invisible } from '@hcaptcha/react-hcaptcha';

function MyForm() {
  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    // The verification will happen automatically
  };

  return (
    <form onSubmit={handleSubmit}>
      <ReCaptchaV2Invisible
        sitekey='your-site-key'
        badge='bottomright'
        onSuccess={(token) => {
          // Handle the verification token
          console.log('Verification successful:', token);
        }}
      />
      <button type='submit'>Submit</button>
    </form>
  );
}
```

## Key Differences

1. **v2 Checkbox**:

   - Visible widget
   - User interaction required
   - Best for forms where you want explicit user verification

2. **v2 Invisible**:

   - Hidden widget
   - Shows challenges only when necessary
   - Good for seamless user experience

3. **v3**:
   - No visible widget
   - Returns a score (0.0 to 1.0)
   - Best for background verification

Choose the appropriate type based on your security requirements and user experience goals. The explicit implementation gives you the flexibility to implement any of these approaches while maintaining full control over the verification process.

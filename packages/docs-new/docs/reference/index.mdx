## Common Types

These types are shared across all packages:

### ReCaptchaType

```typescript
type ReCaptchaType = 'v3' | 'v2-invisible' | 'v2-checkbox';
```

The type of reCAPTCHA to use:

- `v3` - Invisible with risk scoring
- `v2-invisible` - Background verification
- `v2-checkbox` - Interactive checkbox

### ReCaptchaTheme

```typescript
type ReCaptchaTheme = 'light' | 'dark';
```

Visual theme for reCAPTCHA components.

### ReCaptchaSize

```typescript
type ReCaptchaSize = 'compact' | 'normal';
```

Size variant for v2 checkbox components.

### ReCaptchaBadgePosition

```typescript
type ReCaptchaBadgePosition = 'bottomright' | 'bottomleft' | 'inline' | 'hidden';
```

Position of the reCAPTCHA badge for v3 and v2-invisible.

## Configuration Types

### BaseReCaptchaConfig

```typescript
interface BaseReCaptchaConfig {
  siteKey: string;
  language?: ReCaptchaLanguage;
  isEnterprise?: boolean;
  host?: 'google.com' | 'recaptcha.net';
  scriptProps?: React.ScriptHTMLAttributes<HTMLScriptElement>;
  onLoad?(): void;
  onError?(): void;
}
```

Base configuration shared across all reCAPTCHA types.

### V3Config

```typescript
interface V3Config extends BaseReCaptchaConfig {
  type: 'v3';
  explicit?: {
    badge?: ReCaptchaBadgePosition;
    container?: string | HTMLElement;
    tabIndex?: number;
    inherit?: boolean;
    callback?(token: string): void;
    errorCallback?(): void;
    expiredCallback?(): void;
  };
}
```

Configuration specific to reCAPTCHA v3.

### V2InvisibleConfig

```typescript
interface V2InvisibleConfig extends BaseReCaptchaConfig {
  type: 'v2-invisible';
  explicit?: {
    badge?: ReCaptchaBadgePosition;
    container?: string | HTMLElement;
    tabIndex?: number;
    inherit?: boolean;
    callback?(token: string): void;
    errorCallback?(): void;
    expiredCallback?(): void;
  };
}
```

Configuration for reCAPTCHA v2 invisible.

### V2CheckboxConfig

```typescript
interface V2CheckboxConfig extends BaseReCaptchaConfig {
```

## Response Types

### ReCaptchaResponse

```typescript
interface ReCaptchaResponse {
  success: boolean;
  challenge_ts?: string;
  hostname?: string;
  score?: number; // Only for v3
  action?: string; // Only for v3
  'error-codes'?: string[];
}
```

Server-side verification response from Google.

### ReCaptchaError

```typescript
interface ReCaptchaError {
  code: string;
  message: string;
  details?: any;
}
```

Error object returned by reCAPTCHA operations.

## Environment Variables

For secure configuration, use these environment variables:

```bash
# Site keys (public)
NEXT_PUBLIC_RECAPTCHA_SITE_KEY=your_site_key
VITE_RECAPTCHA_SITE_KEY=your_site_key

# Secret keys (server-side only)
RECAPTCHA_SECRET_KEY=your_secret_key

# Enterprise configuration
RECAPTCHA_IS_ENTERPRISE=true
RECAPTCHA_PROJECT_ID=your_project_id
```

## Browser Support

- **Modern Browsers**: Chrome 60+, Firefox 60+, Safari 12+, Edge 79+
- **JavaScript**: ES2018+ features required

### Polyfills

For older browser support, ensure these APIs are available:

- `Promise`
- `fetch` (or use a polyfill)
- `async/await` syntax support

## Migration Guide

### From v1 to v2

Key breaking changes:

1. **Provider Props**: Changed from `reCaptchaKey` to `siteKey`
2. **Hook Returns**: Added loading states and error handling
3. **Type Safety**: Improved TypeScript definitions

```tsx
// v1 (deprecated)
<GoogleReCaptchaProvider reCaptchaKey="key">

// v2 (current)
<GoogleReCaptchaProvider siteKey="key" type="v3">
```

### Configuration Updates

```tsx
// Old configuration
const config = {
  sitekey: 'key',
  theme: 'light'
};

// New configuration
const config = {
  siteKey: 'key',
  type: 'v3',
  explicit: {
    theme: 'light'
  }
};
```
